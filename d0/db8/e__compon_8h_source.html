00134 /* note on _attach_count ...
00135  * The int argument is the initial _attach_count (default = 0)
00136  * Set it to CC_STATIC for static default versions that will never be deleted.
00137  * Set it to 0 (default) for auto versions, so they can be deleted.
00138  * A common will not be deleted on a detach if its _attach_count != 0
00139  * A failed assertion from the common destructor probably means
00140  * the common is being deleted before a device it is attached to is,
00141  * without being first detached.
00142  * This is why ~COMPONENT destroys the subckt explicitly.
00143  *
00144  * Static commons (CC_STATIC) must be in file scope, not in function scope,
00145  * because local statics are deleted first, before any globals.
00146  * //BUG// possible portability problem.  What is deletion order?
00147  */
