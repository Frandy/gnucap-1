00129 /*parse_top_item : To know:
00130  *What does CS& refer to
00131  *  CS stands for command string which is defined in ap.h and various functions   
00132  *  on it are defined in ap*.cc
00133  *what does get_line method on CS& do ?
00134  *  get_line will ‘getlines’ from the file ptr, else if no file, get from 
00135  *  keyboard (stdin) using getcmd (which will get a command, also log, echo etc)
00136  *what is the class CARD_LIST* ?
00137  *      
00138  *what does new__instance of cmd mean?
00139  *      It is defined in u_lang.cc. It does the following :
00140  *      /11/find_type_in_string : specific to each language which gives the type of the  *cmd
00141                 Eg: verilog : comment ,resistor etc
00142  * Then find_proto for the given type.
00143  * Clone the instance : new_instance=proto->clone_instance()
00144  * set the owner of the instance : new_instance->set_owner(owner)
00145  * Then we parse the command using new_instance : x=parse_item(cmd,new_instance)
00146  *              for und. parse_item : // See Stroustrup 15.4.5
00147  *              if MODEL_SUBCKT, return parse_module()
00148  *              elsif COMPONENT, return parse_instance()
00149  *              elsif MODEL_CARD, return parse_paramset()
00150  *              elsif DEV_COMMENT, return parse_comment()
00151  *              elsif DEV_DOT, return parse_command()
00152  *              else return NULL
00153  *      Now Scope->push_back(x) which will add this to the list “Scope._cl”.
00154  */     
00155         
00156 void LANG_GSCHEM::parse_top_item(CS& cmd, CARD_LIST* Scope)

00158   cmd.get_line("gnucap-gschem>");
